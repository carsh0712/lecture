MVP Review

6. MVP 제작과 검수

MVP는 Minimum Viable Product의 약자로, 최소로 실행 가능한 제품이라는 뜻이다. 쉽게 말하면 가장 작은 첫 버전이다. 모든 기능을 넣는 것이 아니라, 실제 업무에서 쓸 수 있는 최소 흐름을 먼저 만들고 확인한다.

MVP 제작 흐름

flowchart TD
  A["핵심 업무 흐름 선택"]
  B["첫 화면과 기능 범위 결정"]
  C["AI Agent가 작은 단위로 구현"]
  D["로컬 또는 미리보기 화면 확인"]
  E{"업무 기준에 맞는가?"}
  F["수정 요청"]
  G["Git/GitHub에 기록"]
  H["배포 또는 공유"]

  A --> B --> C --> D --> E
  E -->|"아니요"| F --> C
  E -->|"예"| G --> H

MVP에 넣을 것과 미룰 것

구분 포함할 것 나중에 할 것
문의 관리 도구 등록, 목록, 상태 변경 알림, 통계, 고객용 화면
매뉴얼 사이트 대시보드, 메뉴, 핵심 문서 검색, 권한, 댓글
파일 정리 자동화 파일 읽기, 변환, 결과 저장 예약 실행, 관리자 화면, 상세 로그

검수 체크리스트

  • 처음 사용자가 무엇을 눌러야 하는지 보이는가?
  • 핵심 기능이 실제 업무 순서와 맞는가?
  • 잘못 입력했을 때 되돌리거나 수정할 수 있는가?
  • 모바일이나 작은 화면에서 버튼과 글자가 겹치지 않는가?
  • 저장된 데이터가 다시 조회되는가?
  • API가 있다면 Swagger/OpenAPI 화면에서 요청과 응답이 업무 흐름과 맞는가?
  • 예상하지 않은 파일이나 설정이 바뀌지 않았는가?

AI Agent에게 말하는 예시

첫 MVP를 만들기 전에 범위를 다시 줄여줘.

목표:
- 운영 담당자가 문의를 등록하고 처리 상태를 바꾸는 첫 버전

포함할 것:
- 문의 등록 화면
- 문의 목록 화면
- 상태 변경 기능

제외할 것:
- 로그인
- 알림
- 통계

구현 후에는 내가 확인할 체크리스트도 함께 작성해줘.
API가 있다면 Swagger/OpenAPI에서 확인할 요청, 응답, 오류 상황도 함께 정리해줘.
