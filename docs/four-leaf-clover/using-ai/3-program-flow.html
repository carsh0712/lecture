Section 3

3 단계. 프로그램 만들기

이 단계에서 이해할 것

프로그램으로 만들기로 결정했다면, 바로 코드를 작성하는 것이 아니라 먼저 흐름을 정리해야 한다. 비개발자는 모든 개발 내용을 알 필요는 없지만, 어떤 순서로 결과물이 만들어지는지는 이해하고 있어야 AI Agent와 더 쉽게 협업할 수 있다.

아래 흐름도는 우리가 관리하는 사이트의 운영 데이터를 가져와 화면에서 확인하고, 필요한 경우 상태를 저장하는 프로그램을 만든다고 가정한 예시다. 

flowchart TD
  A["만들 프로그램의 목적 정리"]
  C["필요한 데이터 확인"]
  D["어떤 개발 플랫폼을 사용할지 결정"]
  F["AI Agent가 프로그램 파일 작성"]
  G["로컬에서 확인"]
  H{"수정이 필요한가?"}
  I["AI Agent에게 수정 요청"]
  J["OSS(Git)에 저장"]
  K["클라우드 서버에 배포"]
  L["운영자가 사용"]
  M["피드백 반영"]

  A --> C --> D --> F --> G --> H
  H -->|"예"| I --> G
  H -->|"아니오"| J --> K --> L --> M --> I

흐름도에서 확인할 것

단계 확인할 내용
만들 프로그램의 목적 정리 무엇을 줄이고 싶은지, 어떤 업무를 편하게 만들고 싶은지 정리한다.
필요한 데이터 확인 화면에 보여줄 항목과 저장해야 할 값을 구분한다.
어떤 개발 플랫폼을 사용할지 결정 웹앱, 서버 프로그램, 내부 도구처럼 어떤 방식으로 만들지 정한다.
AI Agent가 프로그램 파일 작성 AI Agent가 화면, 서버, 설정 파일을 만든다.
로컬에서 확인 배포 전 내 컴퓨터에서 화면, 버튼, 데이터 표시가 맞는지 확인한다.
OSS(Git)에 저장 확인된 파일을 저장소에 올려 변경 이력을 남긴다.
클라우드 서버에 배포 팀이 접속할 수 있도록 서버에 올린다.
운영자가 사용 실제 업무에서 최신 데이터를 확인하고 처리한다.
피드백 반영 사용 중 발견한 불편함을 다시 수정한다.

AI Agent에게 이렇게 말하면 좋다

우리가 관리하는 사이트의 운영 데이터를 확인하는 프로그램을 만들려고 해.
비개발자도 이해할 수 있게 개발 순서를 먼저 정리해줘.

운영자는 매일 문의 목록을 보고, 상태별로 필터링하고, 처리 상태를 변경해야 해.
데이터는 운영 중인 웹사이트에서 최신 상태로 가져와야 해.

먼저 어떤 개발 플랫폼으로 만드는 것이 적합한지 비교하고,
필요한 화면, 데이터 흐름, 저장이 필요한 항목, 테스트 순서, 배포 전 확인 사항을 정리해줘.

중요한 기준

프로그램 개발은 한 번에 완성하는 작업이 아니다. 화면을 만들고, 확인하고, 수정하고, 다시 확인하는 과정이 반복된다. 비개발자는 이 반복 과정에서 업무 기준이 맞는지 확인하는 역할을 맡는다.

핵심은 "AI가 코드를 만든다"가 아니라 "사람이 업무 기준을 설명하고, AI가 실행 가능한 형태로 바꾼다"는 점이다.
