Section 1-4

클라우드 서버 앱 생성 (수정예정)

목적

클라우드 서버 앱은 Git에 올라간 HTML 결과물을 서버에서 실행하고 공유 URL로 제공하기 위한 배포 단위다. 쉽게 말해 “Git에 올린 결과물을 웹에서 볼 수 있게 연결하는 서비스”라고 이해하면 된다.

생성 전 준비물

  • HTML 결과물이 들어 있는 OSS(Git) 리포지토리
  • HTML을 서버로 제공하기 위한 server.js
  • 클라우드 서버가 실행 방법을 알 수 있게 하는 Dockerfile
  • 배포할 브랜치와 경로에 대한 확인

진행 순서

  1. 클라우드 서버에서 새 앱 또는 서비스를 생성한다.
  2. OSS(Git) 리포지토리를 연결한다.
  3. 배포할 브랜치와 Dockerfile 위치를 확인한다.
  4. 환경 설정을 확인한 뒤 배포를 실행한다.
  5. 생성된 URL로 접속해 HTML 화면이 정상적으로 보이는지 확인한다.

확인할 점

  • 클라우드 서버가 연결한 Git 리포지토리가 맞는지 확인한다.
  • 최신 변경 내용이 push되어 있는지 확인한다.
  • 배포 후 화면, 링크, 검색 필터 같은 주요 기능을 직접 확인한다.
클라우드 서버 앱 생성은 보통 최초 1회 작업이다. 이후 HTML 문서를 수정할 때는 Git에 commit/push하고 다시 배포하는 흐름을 반복하면 된다.
