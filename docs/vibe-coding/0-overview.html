Overview

0. AI Agent 프로젝트 전체 흐름

비개발자가 AI Agent로 프로젝트를 만들 때는 코드를 바로 만드는 것보다, 문제를 정의하고 기술 선택을 정리한 뒤 작은 첫 버전을 확인하는 순서가 중요하다. 이 페이지는 전체 흐름을 한눈에 보는 안내서다.

사람과 AI Agent의 역할

단계 비개발자가 결정할 것 AI Agent에게 맡길 것 결과물
프로젝트 정의 누가 어떤 문제를 해결해야 하는지 요구사항을 표와 범위로 정리 프로젝트 브리프
플랫폼 결정 화면 중심인지, 자동화 중심인지 기술 후보와 장단점 비교 추천 플랫폼
데이터베이스 결정 어떤 데이터를 저장하고 얼마나 중요한지 DB 필요 여부와 후보 비교 저장 방식 결정
Git/GitHub 협업 저장소 공개 여부와 변경 승인 기준 변경 파일 정리, 커밋 메시지 제안 안전한 변경 기록
호스팅/배포 비용 누가 접속하고 예산은 얼마인지 배포 후보와 비용 위험 비교 배포 대상 후보
MVP 제작 첫 버전에 꼭 필요한 기능 작은 단위로 구현하고 수정 사용 가능한 첫 버전

읽는 순서

  1. 프로젝트 정의에서 문제와 범위를 먼저 정한다.
  2. 플랫폼 결정에서 어떤 방식으로 만들지 고른다.
  3. 데이터베이스 결정에서 저장 방식이 필요한지 판단한다.
  4. Git/GitHub 협업에서 AI 변경을 안전하게 기록하는 방법을 확인한다.
  5. 호스팅/배포 비용에서 어디에 올릴지와 비용 위험을 비교한다.

전체 요청 프롬프트

비개발자가 AI Agent로 프로젝트를 만들려고 해.
바로 코딩하지 말고, 먼저 전체 진행 계획을 단계별로 정리해줘.

정리할 순서:
1. 프로젝트 정의: 문제, 사용자, 성공 기준, 제외 범위
2. 플랫폼 결정: React, Node.js, Python, 노코드 중 후보 비교
3. 데이터베이스 결정: DB 없음, 파일, SQLite, PostgreSQL 중 후보 비교
4. Git/GitHub 협업: 저장소, 브랜치, 커밋, PR 기준
5. 호스팅/배포 비용: 어디에 올릴지와 비용 위험
6. MVP 범위: 가장 작은 첫 버전의 기능

각 단계마다 사람이 결정할 것, AI가 도와줄 것, 결과물을 표로 정리해줘.
