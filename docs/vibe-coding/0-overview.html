사람과 AI Agent의 역할

단계 비개발자가 결정할 것 AI Agent에게 맡길 것 결과물
프로젝트 정의 누가 어떤 문제를 해결해야 하는지 요구사항을 표와 범위로 정리 프로젝트 브리프
플랫폼 결정 화면 중심인지, 자동화 중심인지 기술 후보와 장단점 비교 추천 플랫폼
데이터베이스 결정 어떤 데이터를 저장하고 얼마나 중요한지 DB 필요 여부와 후보 비교 저장 방식 결정
호스팅/배포 비용 누가 접속하고 예산은 얼마인지 배포 후보와 비용 위험 비교 배포 대상 후보
AI Agent 요청법 무엇을 맡기고 무엇을 확인할지 작업 계획, 구현, 수정 요청을 작게 나누기 명확한 요청 문장
MVP 제작 첫 버전에 꼭 필요한 기능 작은 단위로 구현하고 수정 사용 가능한 첫 버전
Git/GitHub 협업 저장소 공개 여부와 변경 승인 기준 변경 파일 정리, 커밋 메시지 제안 공유 가능한 변경 기록
프롬프트 템플릿 반복해서 쓸 요청 문장 상황별 요청 예시 정리 재사용 가능한 프롬프트
도움이 되는 도구 단계별로 어떤 도구가 필요한지 기획, 구현, 검수, 배포 도구 구분 프로젝트 도구 목록
용어 정리 낯선 개발 용어의 뜻 프로젝트 흐름에 맞춰 쉬운 말로 풀이 비개발자용 용어 사전

읽는 순서

  1. 프로젝트 정의에서 문제와 범위를 먼저 정한다.
  2. 플랫폼 결정에서 어떤 방식으로 만들지 고른다.
  3. 데이터베이스 결정에서 저장 방식이 필요한지 판단한다.
  4. 호스팅/배포 비용에서 어디에 올릴지와 비용 위험을 비교한다.
  5. AI Agent 요청법에서 일을 작게 나누어 맡기는 방식을 익힌다.
  6. MVP 제작과 검수에서 첫 버전을 만들고 확인한다.
  7. Git/GitHub 협업에서 AI 변경을 안전하게 기록하고 공유한다.
  8. 프롬프트 템플릿을 반복 작업에 활용한다.
  9. 도움이 되는 도구에서 단계별 도구를 확인한다.
  10. 용어 정리에서 낯선 단어를 확인한다.

전체 요청 프롬프트

비개발자가 AI Agent로 프로젝트를 만들려고 해.
바로 코딩하지 말고, 먼저 전체 진행 계획을 단계별로 정리해줘.

정리할 순서:
1. 프로젝트 정의: 문제, 사용자, 성공 기준, 제외 범위
2. 플랫폼 결정: React, Node.js, Python, 정적 HTML 중 후보 비교
3. 데이터베이스 결정: DB 없음, 파일, SQLite, PostgreSQL 중 후보 비교
4. 호스팅/배포 비용: 어디에 올릴지와 비용 위험
5. AI Agent 요청법: 일을 작게 나누어 맡기는 방식
6. MVP 제작과 검수: 가장 작은 첫 버전의 기능과 확인 기준
7. Git/GitHub 협업: MVP 변경 내용을 저장소에 기록하고 공유하는 기준
8. 프롬프트 템플릿: 반복해서 쓸 요청 문장
9. 도움이 되는 도구: 기획, 구현, 검수, 배포, 데이터 확인 도구
10. 용어 정리: 비개발자가 알아야 할 개발 용어 풀이

각 단계마다 사람이 결정할 것, AI가 도와줄 것, 결과물을 표로 정리해줘.
