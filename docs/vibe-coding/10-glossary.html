10. 비개발자를 위한 용어 정리

AI Agent로 프로젝트를 만들 때 자주 나오는 단어를 업무 관점에서 풀어 쓴 사전입니다. 정확한 기술 정의보다 “왜 알아야 하는지”와 “언제 쓰는 말인지”를 먼저 이해하는 데 초점을 둡니다.

프로젝트 기획 용어

용어 쉽게 말하면 언제 쓰나
요구사항 프로젝트가 반드시 해내야 하는 일 AI Agent에게 “무엇을 만들어야 하는지” 설명할 때
범위 이번에 만들 것과 만들지 않을 것을 나누는 선 기능이 계속 늘어나지 않게 막을 때
MVP Minimum Viable Product의 약자. 최소로 실행 가능한 제품, 즉 가장 작은 첫 버전 완벽한 제품보다 먼저 써볼 수 있는 버전을 만들 때. Minimum은 최소, Viable은 실제로 쓸 수 있음, Product는 결과물이라는 뜻
사용자 시나리오 사용자가 어떤 순서로 화면을 쓰는지 적은 이야기 버튼, 입력창, 결과 화면의 흐름을 설명할 때
검수 기준 완료 여부를 판단하는 확인 목록 AI가 만든 결과가 업무에 맞는지 확인할 때

화면과 기능 용어

용어 쉽게 말하면 예시
프론트엔드 사용자가 직접 보는 화면 부분 목록, 버튼, 입력창, 대시보드
백엔드 화면 뒤에서 데이터를 처리하는 부분 로그인 확인, 저장, 계산, 권한 검사
API 화면과 서버가 정해진 방식으로 주고받는 약속 “고객 목록을 가져와줘”, “주문을 저장해줘”
Swagger / OpenAPI API를 사람이 읽고 테스트할 수 있게 보여주는 문서 화면과 표준 형식 백엔드 기능이 있을 때 “어떤 요청을 보내면 어떤 결과가 오는지” 확인함
라우팅 주소에 따라 어떤 화면을 보여줄지 정하는 것 /login은 로그인 화면, /dashboard는 대시보드
반응형 PC, 태블릿, 모바일에서 화면이 자연스럽게 바뀌는 것 모바일에서는 메뉴가 접히고 카드가 한 줄로 표시됨

데이터 용어

용어 쉽게 말하면 주의할 점
데이터베이스(DB) 정보를 정리해서 저장하는 공간 고객, 주문, 상담 기록처럼 계속 보관할 정보가 있을 때 필요함
테이블 엑셀 표처럼 같은 종류의 데이터를 모아 둔 곳 고객 테이블, 상품 테이블처럼 목적을 나누어 설계함
스키마 데이터를 어떤 칸과 규칙으로 저장할지 정한 설계도 나중에 바꾸기 어렵기 때문에 초기에 검토가 필요함
권한 누가 어떤 정보를 볼 수 있고 바꿀 수 있는지 정한 규칙 개인정보, 관리자 기능, 결제 정보가 있으면 중요함
백업 문제가 생겼을 때 되돌릴 수 있도록 복사본을 남기는 것 운영 데이터가 생기면 반드시 고려해야 함

Git/GitHub 용어

용어 쉽게 말하면 AI Agent와의 관계
Git 파일 변경 이력을 기록하는 도구 AI가 무엇을 바꿨는지 되돌아볼 수 있게 해줌
GitHub Git 기록을 온라인에 보관하고 공유하는 서비스 사람과 AI 작업 결과를 함께 확인하는 공간이 됨
저장소 프로젝트 파일과 변경 기록이 들어 있는 폴더 AI Agent가 작업하는 기본 단위
커밋 의미 있는 변경을 한 번 저장한 기록 “이번에 무엇을 바꿨는지” 설명하는 단위
브랜치 기존 작업을 건드리지 않고 따로 실험하는 작업 줄기 AI에게 새 기능을 안전하게 맡길 때 유용함
Pull Request(PR) 변경 내용을 합치기 전에 검토해 달라고 올리는 요청 AI가 만든 변경을 사람이 확인하고 승인하는 절차

배포와 운영 용어

용어 쉽게 말하면 확인할 것
배포 내 컴퓨터에서 만든 결과를 다른 사람이 접속할 수 있게 올리는 것 주소, 권한, 비용, 오류 여부
호스팅 웹사이트나 앱을 인터넷에 올려주는 서비스 무료 구간, 사용량 제한, 유료 전환 조건
도메인 사람이 기억하기 쉬운 웹 주소 예: example.com
환경 변수 비밀번호나 설정값처럼 코드에 직접 쓰지 않는 값 API 키, DB 비밀번호는 공개 저장소에 올리면 안 됨
로그 앱에서 무슨 일이 일어났는지 남는 기록 오류 원인을 찾거나 사용 흐름을 확인할 때 필요함

AI Agent에게 물어보는 예시

이 프로젝트 문서에서 개발 용어가 나오면 비개발자가 이해할 수 있게 설명해줘.

정리 방식:
- 용어
- 쉬운 설명
- 프로젝트에서 왜 중요한지
- 내가 확인해야 할 질문

특히 API, Swagger/OpenAPI, 데이터베이스, 배포, Git, 커밋, 브랜치, 환경 변수, MVP를 먼저 설명해줘.
