Section 3-3

3-3 단계. 로컬에서 확인하고 수정하기

이 단계의 목적

로컬 확인은 배포 전에 내 컴퓨터에서 프로그램을 실행해보는 단계다. 실제 운영자가 쓰는 흐름대로 눌러보며 화면과 기능이 맞는지 확인한다.

테스트 서버에 접속하기

AI Agent가 프로그램을 실행하면 브라우저에서 접속할 수 있는 주소를 알려준다. 그 주소를 브라우저 주소창에 붙여 넣으면 테스트 화면을 확인할 수 있다.

  • 내 컴퓨터에서 확인할 때: 보통 http://localhost:5173 같은 주소를 사용한다.
  • 서버 기능을 따로 확인할 때: 보통 http://localhost:3000 같은 주소를 사용한다.
  • 클라우드 서버에 올린 뒤 확인할 때: 배포 플랫폼이 알려주는 https://... 주소를 사용한다.

주소가 열리지 않으면 프로그램이 아직 실행 중인지, AI Agent가 안내한 포트 번호가 맞는지 먼저 확인한다.

확인할 것

  • 화면 문구가 운영자에게 이해하기 쉬운가?
  • 검색과 필터가 기대대로 동작하는가?
  • 상태 변경과 메모 저장이 실제 업무 흐름과 맞는가?
  • 오류가 났을 때 무엇을 해야 하는지 알 수 있는가?

AI Agent에게 요청하기

로컬에서 프로그램을 실행해서 확인할 수 있게 도와줘.
브라우저에서 접속해야 하는 주소도 함께 알려줘.
확인해야 할 화면, 버튼, 데이터 표시, 오류 상황을 체크리스트로 정리해줘.

문제가 발견되면 원인을 설명하고 수정까지 진행해줘.
