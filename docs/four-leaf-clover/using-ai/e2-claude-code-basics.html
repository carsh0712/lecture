Chapter 0

Claude Code 이해하기

Claude Code를 사용할 때 등장하는 것들

Claude Desktop vs Claude Code CLI
Claude Desktop은 UI환경에서 사용하는 AI Agent이다.
Claude Code CLI는 터미널에서 텍스트 환경에서 사용하는 AI Agent이다.
두 화면은 같은 Claude를 다른 방식으로 사용하는 도구라고 이해하면 된다.
MCP 서버
AI Agent가 외부 도구와 연결될 때 사용하는 연결 서버다. 쉽게 말해 GitHub, Confluence, Jira, GQuery 같은 도구에 접근하기 위한 통로라고 보면 된다. 예를 들어 GitHub MCP 서버는 Claude가 GitHub 저장소, 이슈, Pull Request 같은 정보를 작업에 활용할 수 있게 해준다.
스킬
Claude가 특정 업무를 더 잘 수행하도록 정리해둔 작업 지침이다. 반복해서 쓰는 절차, 체크리스트, 업무 방식, 결과물 형식을 묶어둔 가이드에 가깝다.
작업 디렉토리
Claude Code가 기준으로 삼는 작업 폴더다. Claude Code는 보통 특정 프로젝트 폴더에서 시작하며, 그 폴더 안의 파일을 읽고 수정하면서 결과물을 만든다. 비개발자 관점에서는 “AI에게 열어준 작업 범위”라고 이해하면 된다.

실제 업무에서는 어떻게 사용하는가

  1. Claude Code로 작업할 프로젝트 디렉토리를 정한다.
  2. 필요한 MCP 서버를 설치하고 연결한다.
  3. 작업 디렉토리에서 프롬프트를 작성하고 실행한다.
  4. Claude는 연결된 Confluence, GQuery 같은 곳에서 데이터를 가져온다.
  5. Claude는 가져온 데이터를 가공하여 화면에 보여주거나 결과물 파일로 작성한다.
  6. 결과물은 confluence에 작성하거나, OSS(Git)과 클라우드 서버에 배포하여 공유할 수 있다.

비개발자가 알아두면 되는 범위

  • 모든 기술 구조를 외울 필요는 없다.
  • Desktop과 CLI가 같은 Claude Code를 다른 방식으로 사용하는 도구라는 점을 이해한다.
  • 민감정보, 계정 정보, 내부 문서는 임의로 공유하지 않는다.
