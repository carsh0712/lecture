Section 3-5

3-5 단계. 클라우드 서버에 배포하기

이 단계의 목적

클라우드 서버에 배포하는 것은 팀원이 웹 주소로 프로그램에 접속할 수 있게 만드는 단계다. 로컬에서만 실행되던 프로그램을 실제 업무에서 사용할 수 있는 상태로 옮긴다고 이해하면 된다.

배포 전 확인할 것

  • OSS(Git)에 최신 파일이 올라가 있는가?
  • 실행 방법을 알려주는 설정 파일이 준비되어 있는가?
  • 사용할 클라우드 서비스에서 요구하는 배포 방법을 확인했는가?
  • 배포 후 접속할 사람과 공유 범위가 정해져 있는가?

클라우드 서비스마다 설정 화면, 배포 버튼, 필요한 파일 이름이 다를 수 있다. 이 경우에는 사용하는 서비스에 맞는 방법을 직접 찾아 적용해야 한다.

AI Agent에게 요청하기

OSS(Git)에 저장된 프로그램을 클라우드 서버에 배포할 준비를 해줘.
내가 사용할 클라우드 서비스는 [클라우드 서비스 이름]이야.

이 서비스에 맞는 배포 방법을 확인하고, 필요한 설정 파일과 실행 방법을 정리해줘.
서비스별로 추가로 확인해야 하는 공식 문서나 설정 항목이 있으면 함께 알려줘.

배포가 끝나면 접속 URL에서 화면과 주요 기능이 정상 동작하는지 확인해줘.
