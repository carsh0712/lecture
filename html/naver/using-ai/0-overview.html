개요

AI를 활용한 결과물 작성 흐름

AI 사용하기 챕터에서는 프롬프트를 작성한 뒤 Claude Code가 내부 지식 저장소에서 필요한 정보를 가져오고, 그 정보를 목적에 맞게 정리하여 공유 가능한 결과물로 만드는 과정을 다룬다.

아래 흐름도는 기획자와 PM이 전체 작업 순서를 한눈에 이해할 수 있도록 정리한 것이다. 

flowchart TD
  A["Prompt 작성"]
  B["Claude Code가 MCP 서버로
Confluence 또는 CQuery에 연결해서 정보 가져오기"]
  C["Claude Code가 정보 가공"]
  D{"배포 방식 선택"}
  E["Confluence에 작성하기"]
  F["HTML로 작성"]
  G["OSS(git)에 푸시"]
  H["n3r에 서버 연결 및 배포"]

  A --> B --> C --> D
  D -->|"방식 1"| E
  D -->|"방식 2"| F --> G --> H

용어 정리

MCP 서버
Claude code가 외부 도구와 연결될 때 사용하는 연결 서버다. 쉽게 말해 Confluence, Jira, CQuery 같은 도구에 접근하기 위한 통로라고 보면 된다.

배포 방식

Confluence에 작성하기
wiki 문서로 바로 공유하고 리뷰하기 좋은 방식이다.
HTML로 작성 후 배포하기
Claude Code가 작성한 HTML 파일을 n3r 서버를 통해 웹 페이지 형태로 공유하는 방식이다. Confluence에 작성하는 것보다 자유도가 높고, 여러가지 효과(검색 필터 등)를 넣을 수 있는 장점이 있다.
