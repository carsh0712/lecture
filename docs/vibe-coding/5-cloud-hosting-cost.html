5. 호스팅/배포 비용 정리

이 문서는 바이브코딩으로 만든 프로젝트를 어디에 올릴지 결정할 때 비용을 대략 비교하기 위한 자료다. 가격은 2026년 7월 5일에 공식 가격 페이지를 확인한 기준이며, 실제 결제 전에는 각 서비스의 최신 가격과 환율, 세금, 사용량 조건을 다시 확인해야 한다.

먼저 결정할 것

호스팅 비용은 "어느 회사가 싼가"보다 "내 프로젝트가 무엇을 필요로 하는가"에 따라 달라진다. 단순한 안내 페이지와 로그인, 데이터베이스, 파일 업로드가 있는 업무 도구는 필요한 비용 구조가 다르다.

비용 비교표

아래 금액은 대표적인 시작 비용을 이해하기 위한 요약이다. 대부분의 서비스는 무료 구간이 있어도 트래픽, 빌드 시간, 함수 실행, 데이터 저장량, 팀 기능, 도메인, 로그 보관 기간에 따라 비용이 달라질 수 있다.

서비스 어울리는 경우 시작 비용 감각 주의할 비용
GitHub Pages 정적 HTML 문서, 매뉴얼, 소개 페이지 GitHub Free 플랜은 $0/월이고, GitHub Pages는 저장소 기반으로 사이트를 호스팅한다. 서버 기능과 데이터베이스가 필요한 앱에는 맞지 않는다. 빌드 자동화 사용량은 GitHub Actions 조건을 확인한다.
Cloudflare Pages 정적 사이트, 빠른 전세계 배포, 간단한 프론트엔드 공식 페이지에서 무료 시작을 제공한다. 신용카드 없이 시작 가능하다고 안내한다. Workers, D1, R2, KV 같은 추가 기능을 쓰면 별도 사용량 비용을 확인해야 한다.
Vercel React, Next.js, 프론트엔드 중심 웹앱 Hobby $0/월, Pro $20/월부터 시작한다. Pro에는 포함 사용량이 있지만, 요청 수, 데이터 전송량, 함수 실행, 이미지 최적화 등 초과 사용량 비용이 붙을 수 있다.
Netlify 정적 사이트, 배포 미리보기, 간단한 서버리스 기능 Free $0, Personal $9/월, Pro $20/월부터 시작한다. 크레딧 방식으로 배포, 컴퓨트, 대역폭, 요청 수가 계산된다. 자동 충전 설정 여부를 확인한다.
Render 서버 API, 백그라운드 작업, PostgreSQL까지 한곳에서 관리 정적 사이트는 $0/월부터 가능하다. 웹 서비스는 Free 또는 Starter $7/월부터, 유료 작업공간은 Pro $25/월 + compute 구조다. Postgres 무료는 30일 제한이 있으며, 운영 DB는 Basic $6/월 또는 그 이상을 검토한다. 대역폭 초과도 확인한다.
Railway 작은 풀스택 앱, DB 포함 빠른 실험 Free는 30일 체험 크레딧 후 조건을 확인해야 하며, Hobby는 $5 최소 사용량부터 시작한다. 사용량 기반 과금이라 앱을 켜 둔 시간, CPU, 메모리, 스토리지, 네트워크 사용량을 확인해야 한다.
Supabase PostgreSQL DB, 로그인, 파일 저장소가 필요한 앱 Free 플랜은 두 개의 무료 프로젝트를 제공한다. 유료 플랜은 고정 구독료와 사용량 비용이 함께 계산된다. DB 크기, egress, 인증 사용자 수, 스토리지, Edge Function 호출량이 늘어나면 초과 비용이 발생할 수 있다.
AWS Lightsail 서버를 직접 운영할 수 있고 월 고정비를 선호하는 경우 Linux/Unix public IPv4 번들은 $5/월부터, IPv6 전용 번들은 $3.50/월부터 시작한다. 서버 보안, 업데이트, 백업을 직접 챙겨야 한다. DB, 로드밸런서, 스토리지, 초과 트래픽은 별도 비용이 된다.

상황별 추천

상황 우선 검토 이유
매뉴얼, 랜딩 페이지, 회사 내부 안내 페이지 GitHub Pages, Cloudflare Pages 서버와 DB가 없으면 무료 또는 낮은 비용으로 시작하기 쉽다.
React나 Next.js로 만든 프론트엔드 앱 Vercel, Netlify, Cloudflare Pages Git 연결과 자동 배포가 편하고, 미리보기 링크를 만들기 쉽다.
API 서버가 필요한 작은 업무 도구 Render, Railway Node.js, Python 같은 서버 앱을 비교적 간단히 올릴 수 있다.
로그인과 데이터 저장이 필요한 앱 Supabase + Vercel 또는 Supabase + Netlify 프론트엔드와 DB를 분리하면 비개발자도 비용 항목을 나누어 이해하기 쉽다.
운영자가 서버 관리 경험이 있는 프로젝트 AWS Lightsail 월 고정비 예측은 쉽지만 서버 보안과 장애 대응 책임이 커진다.

숨은 비용 체크리스트

  • 도메인 구입 비용이 별도로 필요한가?
  • 서버 기능, 함수 실행, 예약 작업이 무료 구간을 넘을 가능성이 있는가?
  • 데이터베이스 저장 용량과 백업 비용이 포함되어 있는가?
  • 파일 업로드, 이미지 저장, 다운로드 트래픽이 많은가?
  • 팀원이 늘어나면 사용자당 요금이 붙는가?
  • 무료 DB가 일정 기간 후 삭제되거나 일시 정지되는 조건이 있는가?
  • 예산 알림, 지출 제한, 자동 충전 끄기 같은 보호 장치가 있는가?
처음에는 "무료로 시작 가능한가"보다 "유료로 넘어갈 때 어떤 항목에서 비용이 늘어나는가"를 확인하는 것이 중요하다. 특히 DB, 파일 저장소, 트래픽, 팀 기능은 예상보다 빨리 비용 항목이 될 수 있다.

AI Agent에게 물어볼 예시

우리가 만들 프로젝트의 클라우드 호스팅 비용을 비교해줘.

프로젝트 조건:
- 사용자는 사내 직원 5명이다.
- React 화면과 간단한 Node.js API가 필요하다.
- 로그인은 필요하지만 외부 고객에게 공개하지 않는다.
- 하루 데이터 입력은 100건 이하이고, 파일 업로드는 거의 없다.
- 월 비용은 가능하면 3만 원 이하로 시작하고 싶다.

비교해줄 항목:
- GitHub Pages, Vercel, Netlify, Render, Railway, Supabase, AWS Lightsail 중 어떤 조합이 맞는지
- 무료로 시작할 수 있는지
- 유료로 넘어가는 조건
- 숨은 비용 위험
- 비개발자가 운영하기 쉬운 추천안
