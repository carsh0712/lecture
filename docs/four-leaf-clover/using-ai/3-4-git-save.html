Section 3-4

3-4 단계. OSS(Git)에 저장하기

이 단계의 목적

OSS(Git)에 저장하는 것은 완성된 파일을 안전한 저장소에 올리고 변경 이력을 남기는 단계다. 이후 배포나 수정 작업은 이 저장소를 기준으로 진행된다.

저장 전 확인할 것

  • 로컬에서 화면과 기능을 확인했는가?
  • 불필요한 임시 파일이나 민감정보가 포함되지 않았는가?
  • 어떤 내용을 변경했는지 설명할 수 있는가?

AI Agent에게 요청하기

현재 프로그램 파일을 OSS(Git)에 저장할 준비를 해줘.
변경된 파일을 확인하고, 민감정보가 포함되지 않았는지 점검해줘.

문제가 없으면 어떤 변경을 저장하는지 요약하고 commit/push까지 진행해줘.
