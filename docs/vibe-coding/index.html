비개발자를 위한 AI Agent 프로젝트 로드맵

이 매뉴얼은 비개발자가 AI Agent와 함께 프로젝트를 시작하고, 기술 선택을 정리하고, GitHub에 저장하고, 배포까지 이어 가는 전체 흐름을 단계별로 안내합니다.

Step 0

전체 흐름

AI Agent와 프로젝트를 만들 때 어떤 순서로 결정하고 확인해야 하는지 큰 그림을 봅니다.

Step 1

프로젝트 정의

문제, 사용자, 성공 기준, 제외 범위를 먼저 정리해 AI가 엉뚱한 방향으로 가지 않게 합니다.

열기

Step 2

플랫폼 결정

React, Node.js, Python, 정적 HTML 중 프로젝트 성격에 맞는 시작점을 고릅니다.

열기

Step 3

데이터베이스 결정

DB 없음, 파일, SQLite, PostgreSQL 중 데이터 중요도와 사용 방식에 맞는 저장 방법을 고릅니다.

열기

Step 4

호스팅/배포 비용

완성된 프로젝트를 어디에 올릴지, 무료 구간과 유료 전환 지점을 기준으로 비교합니다.

열기

Step 5

AI Agent 요청법

AI에게 일을 작게 나누어 맡기고 결과를 확인하는 요청 방식을 익힙니다.

Step 6

MVP 제작과 검수

가장 작은 첫 버전을 만들고, 화면과 기능을 업무 기준으로 확인합니다.

열기

Step 7

Git/GitHub 협업

AI가 바꾼 파일을 Git으로 기록하고 GitHub로 공유하는 안전한 협업 흐름을 이해합니다.

Step 8

프롬프트 템플릿

프로젝트 정의, 기술 비교, MVP 구현, 수정 요청에 바로 쓸 수 있는 문장을 모아 둡니다.

열기

Step 9

도움이 되는 도구

기획, 화면 구상, AI 구현, GitHub 협업, Swagger/API 확인, 테스트, 배포, 데이터 확인 도구를 정리합니다.

용어 정리

API, Swagger/OpenAPI, DB, 배포, Git, 커밋, 브랜치처럼 비개발자에게 낯선 단어를 쉬운 말로 풀이합니다.
