Section 1-2

HTML로 작성해서 클라우드 서버로 배포하기

목적

HTML로 작성해서 클라우드 서버로 배포하는 방식은 결과물을 웹 페이지처럼 보여주고 싶을 때 사용한다. 검색 필터, 표, 카드 목록, 안내 페이지처럼 화면 구성의 자유도가 필요한 자료에 적합하다.

전체 흐름 한눈에 보기

비개발자 관점에서는 HTML 파일, Node.js, Dockerfile, Git, 클라우드 서버가 각각 다른 역할을 한다고 이해하면 된다. Claude Code는 화면 파일과 실행 준비 파일을 만들고, Git은 그 파일을 보관하며, 클라우드 서버는 Git에 올라간 파일을 서버로 실행한다.

처음 배포할 때는 server.js, Dockerfile, OSS(Git) 리포지토리, remote 연결, 클라우드 서버 생성이 필요하다. 이후 문서를 수정할 때는 HTML 작성 후 다시 commit/push와 배포 흐름을 반복하면 된다.

진행 순서

  1. Claude Code에게 결과물을 HTML 파일로 작성하게 한다.
  2. 로컬에서 HTML 화면과 링크가 의도대로 동작하는지 확인한다.
  3. 최초 1회로 server.js와 Dockerfile을 준비한다.
  4. OSS(Git) 리포지토리를 만들고, 작업 폴더와 remote를 연결한다.
  5. 작성된 HTML, CSS, server.js, Dockerfile 등 필요한 파일을 Git에 commit/push한다.
  6. 최초 1회로 클라우드 서버를 생성한다.
  7. 클라우드 서버에서 해당 서버로 배포한다.
클라우드 서버로 배포하려면 Git에 올라간 파일을 기준으로 서버가 만들어진다. 따라서 HTML 결과물을 로컬에만 두는 것이 아니라, 반드시 OSS(Git)에 commit/push하는 과정이 필요하다.

Dockerfile이 필요한 이유

클라우드 서버에서 배포하려면 서버가 어떤 방식으로 실행될지 알려주는 설정이 필요하다. 이때 사용하는 대표적인 파일이 Dockerfile이다. Dockerfile은 “이 프로젝트를 어떤 환경에서, 어떤 명령으로 실행할지”를 적어두는 배포 설명서라고 이해하면 된다.

단순 HTML 파일이라도 클라우드 서버에서는 서버 형태로 실행되어야 한다. 따라서 HTML 파일을 그대로 열어보는 방식이 아니라, Node.js 같은 실행 환경을 사용해 정적 파일을 제공하는 서버 형태로 실행시키는 구성이 필요하다.

프롬프트 예시

이 내용을 클라우드 서버에 배포할 수 있는 HTML 페이지로 작성해줘.
비개발자가 읽기 쉽게 구성하고, 검색이나 필터가 필요한 부분은 화면에서 사용할 수 있게 만들어줘.
배포를 위해 Git에 커밋할 파일 목록을 정리하고, 클라우드 서버에서 실행 가능한 Node.js 기반 정적 서버 구성과 Dockerfile도 함께 준비해줘.

장점

  • 문서보다 더 자유로운 화면 구성과 인터랙션을 만들 수 있다.
  • 검색, 필터, 카드형 목록 등 사용자가 직접 탐색하는 자료에 적합하다.
  • Git에 기록이 남아 변경 이력을 추적하기 좋다.

단점

  • Git commit/push, 클라우드 서버 배포 같은 추가 과정이 필요하다.
  • Dockerfile과 Node.js 실행 방식처럼 최소한의 배포 구조를 이해해야 한다.
  • 문서 내용만 빠르게 고치는 작업은 Confluence보다 번거로울 수 있다.
