Project Definition

1. 프로젝트 정의하기

AI Agent에게 바로 만들라고 요청하기 전에, 먼저 어떤 문제를 해결할지 정리해야 한다. 이 단계가 분명해야 플랫폼, 데이터베이스, Git, 배포 결정도 흔들리지 않는다.

프로젝트 브리프 항목

항목 질문 예시
문제 무엇이 필요하고, 왜 필요한가? 문의 처리 상태를 한곳에서 확인하고 누락을 줄이는 도구가 필요하다.
사용자 누가 이 도구를 사용할 것인가? 사내 운영 담당자 5명
입력 데이터 사용자가 무엇을 입력하는가? 문의 제목, 고객명, 담당자, 처리 상태
출력 결과 사용자가 무엇을 확인해야 하는가? 상태별 문의 목록과 처리 지연 건수
성공 기준 첫 버전이 쓸 만하다고 판단하는 기준은 무엇인가? 운영자가 문의를 등록하고 상태를 변경할 수 있다.
제외 범위 이번 버전에서 하지 않을 것은 무엇인가? 고객용 로그인, 알림톡 발송, 복잡한 통계

AI Agent에게 말하는 예시

바로 코드를 만들지 말고, 먼저 프로젝트 정의를 정리해줘.

상황:
- 우리는 사내 문의 처리 상태를 더 쉽게 관리하고 싶다.
- 현재는 엑셀로 관리해서 누락과 중복이 생긴다.
- 사용자는 운영 담당자 5명이다.
- 첫 버전에서는 문의 등록, 목록 조회, 상태 변경만 있으면 된다.
- 고객 로그인, 외부 공개, 알림 발송은 이번 범위에서 제외한다.

정리해줄 내용:
- 프로젝트 목적
- 사용자
- 입력 데이터와 출력 결과
- MVP 기능 범위
- 이번 버전에서 제외할 것
- 다음 단계에서 결정해야 할 플랫폼과 DB 질문

완료 기준

  • 프로젝트 목적을 한 문장으로 설명할 수 있다.
  • 사용자가 누구인지 분명하다.
  • 첫 버전에 꼭 필요한 기능과 나중에 할 기능이 구분되어 있다.
  • AI Agent에게 전달할 브리프가 준비되어 있다.
