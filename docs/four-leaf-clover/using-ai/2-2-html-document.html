Section 2-2

2-2 단계. HTML 문서로 작성하기

언제 HTML 문서를 선택할까?

HTML 문서는 내용을 웹 페이지처럼 보여주고 싶을 때 적합하다. 특정 사이트의 운영 방침을 단순한 글이 아니라 검색, 필터, 그래프, 카드 목록처럼 읽기 쉬운 화면으로 제공하고 싶다면 HTML 문서가 좋은 선택이다.

단, HTML 문서는 기본적으로 작성된 내용을 보여주는 문서다. 문의 건수나 처리 상태처럼 계속 바뀌는 데이터를 실시간으로 반영해야 한다면 자동화 프로그램을 고려해야 한다.

HTML 문서에 넣기 좋은 기능

기능 운영 방침 문서에서의 활용
검색 운영자가 "문의", "게시물", "예외" 같은 단어로 필요한 기준을 빠르게 찾을 수 있다.
필터 업무 유형, 담당 부서, 중요도 같은 조건으로 운영 기준을 나누어 볼 수 있다.
그래프 문의 유형 비율, 월별 처리 건수, 예외 상황 발생 현황을 한눈에 보여줄 수 있다.
접기/펼치기 자주 보는 핵심 기준은 먼저 보여주고, 상세 절차는 필요할 때만 펼쳐볼 수 있다.
카드와 강조 영역 긴 문서를 작은 단위로 나누어 운영자가 중요한 내용을 빠르게 훑어볼 수 있다.

AI Agent에게 요청하기

HTML 문서를 요청할 때는 내용뿐 아니라 화면에서 어떤 기능이 필요할지도 함께 말하는 것이 좋다. 그러면 AI Agent가 문서 구조와 화면 기능을 함께 설계할 수 있다. 

특정 사이트의 운영 방침을 HTML 문서로 만들어줘.
비개발자 운영 담당자가 읽을 문서이고, 웹 페이지처럼 보기 쉽게 구성해줘.

문서에는 게시물 관리 기준, 사용자 문의 처리 방식, 관리자 확인 절차, 예외 상황 대응 방법을 포함해줘.
검색 기능, 유형별 필터, 접기/펼치기 영역을 넣어줘.

운영 현황을 설명하는 예시 데이터가 있다면 문의 유형 비율이나 처리 현황을 그래프로 보여줄 수 있게 구성해줘.
단, 실시간으로 바뀌는 데이터가 필요한 부분은 자동화 프로그램으로 확장해야 한다고 표시해줘.

OSS(Git)에 저장하는 이유

HTML 문서는 파일로 만들어지기 때문에 저장 위치가 필요하다. OSS(Git) 저장소에 올려두면 문서 파일을 잃어버리지 않고, 누가 언제 어떤 내용을 수정했는지도 확인할 수 있다.

  • HTML, CSS, 이미지 같은 문서 파일을 한곳에 보관할 수 있다.
  • 수정 이력이 남아 이전 버전과 비교하기 쉽다.
  • 단순 HTML 문서는 Page(OSS/Git)를 통해 링크로 공유할 수 있다.
  • 나중에 클라우드 서버 배포가 필요해져도 같은 저장소를 이어서 사용할 수 있다.

HTML 문서로 충분하지 않은 경우

HTML 문서는 보기 좋은 안내 화면을 만드는 데 강하지만, 계속 바뀌는 데이터를 직접 가져오거나 사용자가 입력한 내용을 저장하는 데는 한계가 있다. 이런 경우에는 자동화 프로그램으로 방향을 바꾸는 것이 좋다.

  • 문의 상태가 매일 바뀌고 최신 데이터가 자동으로 반영되어야 하는 경우
  • 운영자가 화면에서 처리 상태를 변경하고 저장해야 하는 경우
  • 사용자 입력을 받아 데이터베이스나 외부 시스템에 기록해야 하는 경우
  • 권한에 따라 볼 수 있는 데이터가 달라져야 하는 경우
