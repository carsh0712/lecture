AI를 활용한 결과물 작성 흐름

AI 사용하기 챕터에서는 프롬프트를 작성한 뒤 Claude Code가 내부 지식 저장소에서 필요한 정보를 가져오고, 그 정보를 목적에 맞게 정리하여 공유 가능한 결과물로 만드는 과정을 다룬다.

아래 흐름도는 기획자와 PM이 전체 작업 순서를 한눈에 이해할 수 있도록 정리한 것이다. 

flowchart TD
  A["Prompt 작성"]
  B["Claude Code가 MCP 서버로
Confluence 또는 CQuery에 연결해서 정보 가져오기"]
  C["Claude Code가 정보 가공"]
  D{"배포 방식 선택"}
  E["Confluence에 작성하기"]
  F["HTML로 작성"]
  G["OSS(git)에 푸시"]
  H["n3r에 서버 연결 및 배포"]

  A --> B --> C --> D
  D -->|"방식 1"| E
  D -->|"방식 2"| F --> G --> H

단계별 의미

  1. Prompt 작성: AI에게 요청할 업무 목적, 필요한 자료, 원하는 결과물 형태를 정리한다.
  2. MCP 서버 연결: Claude Code가 Confluence 또는 CQuery 같은 내부 도구에서 필요한 정보를 가져온다.
  3. 정보 가공: 가져온 자료를 검토, 요약, 재구성하여 문서나 HTML 결과물로 만들 준비를 한다.
  4. 배포 방식 선택: 결과물을 Confluence 문서로 남길지, HTML로 만들어 서버에 배포할지 결정한다.

배포 방식

  • Confluence에 작성하기: 팀 문서로 바로 공유하고 리뷰하기 좋은 방식이다.
  • HTML로 작성 후 배포하기: OSS(git)에 푸시한 뒤 n3r에 서버를 연결해 웹 페이지 형태로 공유하는 방식이다.
