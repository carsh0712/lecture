8. 프롬프트 템플릿 모음

아래 템플릿은 비개발자가 AI Agent에게 바로 붙여 넣어 사용할 수 있는 요청 예시다. 대괄호 안의 내용만 프로젝트 상황에 맞게 바꾸면 된다.

프로젝트 정의 요청

바로 코딩하지 말고 프로젝트 정의부터 정리해줘.

상황:
- 필요한 것: [무엇이 필요한가]
- 사용자: [누가 사용할까]
- 현재 방식: [지금은 어떻게 처리하나]
- 목표 결과: [무엇이 쉬워져야 하나]

정리해줄 내용:
- 프로젝트 목적
- 사용자
- MVP 범위
- 제외할 것
- 다음에 결정해야 할 질문

플랫폼 비교 요청

우리 프로젝트에 맞는 플랫폼을 비교해줘.

조건:
- 화면 필요 여부: [예/아니요]
- 서버 기능 필요 여부: [예/아니요]
- 데이터 처리/자동화 필요 여부: [예/아니요]
- 사용자 수: [예상 사용자 수]

React, Node.js, Python, 정적 HTML 중 어떤 선택이 맞는지 비교하고 추천해줘.

데이터베이스 비교 요청

이 프로젝트에 데이터베이스가 필요한지 판단해줘.

저장할 데이터:
- [데이터 항목 1]
- [데이터 항목 2]
- [데이터 항목 3]

사용 방식:
- 여러 사람이 동시에 쓰는가: [예/아니요]
- 검색이 필요한가: [예/아니요]
- 백업이 필요한가: [예/아니요]

DB 없음, 파일, SQLite, PostgreSQL 중 추천해줘.

MVP 구현 요청

첫 MVP만 작게 만들어줘.

포함할 기능:
- [기능 1]
- [기능 2]
- [기능 3]

제외할 기능:
- [나중에 할 기능 1]
- [나중에 할 기능 2]

작업 후에는 내가 확인할 검수 체크리스트도 작성해줘.

수정 요청

아래 부분만 수정해줘.

수정할 위치:
- [화면/기능/문서 위치]

현재 문제:
- [무엇이 기대와 다른가]

원하는 결과:
- [어떻게 바뀌어야 하는가]

수정 후 확인할 것:
- [확인 기준]

Git 커밋 전 확인 요청

커밋하기 전에 변경 내용을 비개발자가 이해할 수 있게 정리해줘.

확인할 것:
- 어떤 파일이 바뀌었는지
- 왜 바뀌었는지
- 의도하지 않은 변경이 있는지
- 민감한 정보가 포함되었는지
- 추천 커밋 메시지
