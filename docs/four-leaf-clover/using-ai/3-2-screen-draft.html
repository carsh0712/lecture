Section 3-2

3-2 단계. 화면 초안과 기능 구현하기

이 단계의 목적

화면 초안은 운영자가 무엇을 보고 어떤 버튼을 누를지 먼저 확인하는 단계다. 초안이 괜찮다면 같은 흐름에서 AI Agent에게 실제 기능 구현까지 이어서 요청한다.

AI Agent에게 요청하기

화면 초안을 만들 때는 사용할 개발 방식과 참고할 자료 위치를 함께 알려주는 것이 좋다. 예를 들어 팀 공유 드라이브에 운영 방침 문서가 있다면, 그 자료를 기준으로 첫 화면을 만들어 달라고 요청한다. 

Node.js + Express.js 서버 + React Vite 조합으로 운영 관리 프로그램의 첫 화면 초안을 만들어줘.

자료는 팀 공유 드라이브의 [공유 드라이브 경로 또는 폴더명]에 있는 운영 방침 문서와 처리 예시 파일을 참고해줘.
비개발자 운영 담당자가 사용할 프로그램이야.

첫 단계에서는 완성된 기능보다 화면 구성과 데이터 흐름을 먼저 확인하고 싶어.
어떤 화면을 만들었는지, 어떤 파일이 생성되었는지, 실행하려면 무엇을 해야 하는지도 짧게 설명해줘.

공유 드라이브 자료를 사용할 때는 AI Agent가 실제로 접근할 수 있는 위치를 알려줘야 한다. 민감한 정보가 있다면 그대로 붙여 넣기보다 필요한 항목만 정리해서 전달한다.

기능 구현까지 이어가기

화면 구성이 맞다면 다음에는 같은 대화 안에서 기능을 실제 파일로 만들도록 요청한다. 이때도 코드 설명보다 “무엇이 만들어졌고, 어떻게 확인하면 되는지”를 함께 요청하는 것이 좋다. 

앞에서 만든 화면 초안을 기준으로 실제 기능을 구현해줘.

Node.js + Express.js 서버에서는 공유 드라이브 자료를 읽어오는 흐름을 만들고,
React Vite 화면에서는 그 자료를 운영자가 확인하고 사용할 수 있게 보여줘.

처음에는 완벽한 자동화보다 동작을 확인할 수 있는 수준으로 만들어줘.
작업 후에는 어떤 파일을 만들었는지, 각 파일이 어떤 역할인지, 내가 로컬에서 어떻게 실행해 확인하면 되는지 설명해줘.

수정을 이어 나가기

처음 만든 화면과 기능은 완성본이 아니라 대화의 시작점이다. 화면을 확인한 뒤 불편한 부분, 빠진 정보, 표현이 어색한 부분을 짧게 나누어 계속 요청한다. 

방금 만든 화면과 기능을 기준으로 계속 수정해줘.

수정 요청:
- [예: 운영 방침 문서에서 가져온 항목명을 화면에 더 잘 보이게 정리해줘]
- [예: 공유 드라이브 자료를 다시 확인해서 빠진 처리 기준이 있으면 추가해줘]
- [예: 사용자가 먼저 봐야 하는 정보를 화면 위쪽으로 옮겨줘]

수정 후에는 변경된 화면 영역과 내가 확인해야 할 점을 짧게 정리해줘.
