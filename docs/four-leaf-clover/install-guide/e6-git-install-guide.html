Git 설치 가이드 (수정예정)

Git이 필요한 이유

Git은 HTML 결과물의 변경 이력을 저장하고 OSS(Git) 리포지토리에 업로드하기 위해 필요한 도구다. 클라우드 서버 배포는 Git에 올라간 파일을 기준으로 진행되므로, 로컬 PC에서 Git을 사용할 수 있어야 한다.

설치 여부 확인하기

  1. 터미널을 연다.
  2. 아래 명령어를 입력한다.
git --version

git version 2.40.0처럼 버전이 보이면 Git이 설치된 상태다. 숫자는 PC마다 다를 수 있다.

설치 후 확인할 것

  • 터미널에서 git --version이 정상적으로 동작하는지 확인한다.
  • OSS(Git) 계정에 접근할 수 있는지 확인한다.
  • 회사 보안 정책에 맞는 인증 방식이 필요한지 확인한다.

OSS(Git) 리포지토리 생성

OSS(Git) 리포지토리는 HTML 결과물과 배포 설정 파일을 올려두는 저장소다. 클라우드 서버는 이 저장소에 올라간 파일을 기준으로 서버를 만들고 배포하므로, HTML 배포 전에 리포지토리를 준비해야 한다.

  1. OSS(Git)에서 새 리포지토리를 생성한다.
  2. 리포지토리 이름과 공개 범위, 접근 권한을 확인한다.
  3. 로컬 작업 폴더에서 Git 초기화 여부를 확인한다.
  4. remote를 연결해 로컬 작업 폴더와 OSS(Git) 리포지토리를 연결한다.
  5. 초기 파일을 commit/push해서 클라우드 서버가 가져갈 수 있게 준비한다.

기본 사용 흐름

  1. Claude Code가 만든 HTML, server.js, Dockerfile을 확인한다.
  2. 변경 내용을 commit한다.
  3. commit한 내용을 push해서 OSS(Git)에 올린다.
  4. 클라우드 서버에서 Git에 올라간 최신 파일을 기준으로 배포한다.
commit은 “이번 변경을 저장한다”는 의미이고, push는 “저장한 변경을 OSS(Git)에 올린다”는 의미다. 클라우드 서버 배포는 Git에 push된 파일을 기준으로 진행된다.

Claude Code에게 요청할 수 있는 일

현재 변경된 파일을 확인하고, 클라우드 서버 배포에 필요한 파일만 정리해줘.
문제가 없으면 Git에 commit할 메시지를 추천하고, commit/push 전에 확인해야 할 사항을 알려줘.

주의할 점

  • 계정 정보, 토큰, 비밀번호 같은 민감정보가 파일에 들어가지 않았는지 확인한다.
  • 배포에 필요한 파일만 commit한다.
  • push 후에는 클라우드 서버가 보는 파일도 바뀔 수 있으므로 변경 내용을 한 번 더 확인한다.
