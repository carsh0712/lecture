2. 플랫폼 결정하기

플랫폼은 프로젝트를 어떤 방식으로 만들고 실행할지 정하는 선택이다. 비개발자는 기술 이름을 외우기보다, 만들려는 결과가 화면 중심인지, 서버 기능 중심인지, 데이터 처리 중심인지부터 판단하면 된다.

주요 선택지

플랫폼 쉽게 말하면 어울리는 경우 주의할 점
React 웹 화면을 만드는 도구다. 관리자 화면, 대시보드, 입력 폼처럼 사용자가 보는 화면이 중요한 프로젝트 데이터 저장이나 로그인은 별도 서버나 DB와 함께 설계해야 한다.
Node.js 웹앱의 서버 기능을 만드는 도구다. React 화면 뒤에서 데이터 조회, 저장, API 연결이 필요한 프로젝트 서버 배포, 환경 변수, 보안 설정을 함께 관리해야 한다.
Python 데이터 처리와 자동화에 강한 도구다. 엑셀 정리, 파일 변환, 반복 업무 자동화, 데이터 분석이 중요한 프로젝트 웹 화면까지 필요하면 별도 화면 도구와 연결해야 할 수 있다.

처음에는 이렇게 고른다

  • 화면이 중요하면 React를 먼저 검토한다.
  • 화면과 데이터 저장이 모두 필요하면 React + Node.js 조합을 검토한다.
  • 반복 업무나 파일 처리가 중심이면 Python을 검토한다.
  • 문서나 안내 화면처럼 고정된 내용이면 정적 HTML로 작게 시작할 수 있다.
  • 처음부터 완벽한 기술을 고르기보다, MVP를 가장 빨리 확인할 수 있는 선택을 우선한다.
