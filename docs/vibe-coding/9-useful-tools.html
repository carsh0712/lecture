9. 프로젝트 진행에 도움이 되는 도구

비개발자가 바이브 코딩으로 프로젝트를 진행할 때 모든 도구를 배울 필요는 없다. 단계마다 필요한 도구의 역할만 이해하고, AI Agent에게 무엇을 맡길지 구분하면 된다.

도구를 고르는 흐름

flowchart TD
  A["아이디어와 요구사항 정리"]
  B["화면과 흐름 설명"]
  C["AI Agent로 구현"]
  D["코드 저장과 협업"]
  E["화면과 기능 확인"]
  F["배포와 공유"]
  G["데이터 확인"]

  A --> B --> C --> D --> E --> F --> G

단계별 추천 도구

단계 도구 예시 언제 쓰나 비개발자가 확인할 것
기획/정리 Notion, Confluence, Google Docs 요구사항, 회의 내용, 결정 사항을 정리할 때 목적, 사용자, 제외 범위가 문서에 남아 있는가?
화면 구상 Figma, FigJam, Excalidraw 화면 배치, 버튼 위치, 사용자 흐름을 설명할 때 AI Agent가 화면을 오해하지 않을 정도로 흐름이 보이는가?
AI 구현 Codex, Claude Code, Cursor 파일 생성, 코드 수정, 설명, 리팩터링을 맡길 때 AI가 바꾼 파일과 변경 목적을 설명할 수 있는가?
저장/협업 Git, GitHub 변경 기록을 남기고 팀과 공유할 때 커밋 메시지와 변경 파일이 업무 목적과 맞는가?
테스트/확인 브라우저 개발자 도구, Playwright, Lighthouse 화면 오류, 버튼 동작, 반응형 화면, 성능을 확인할 때 사용자가 실제 업무 순서대로 사용할 수 있는가?
배포/호스팅 GitHub Pages, Vercel, Netlify, Render, Railway 다른 사람이 접속할 수 있게 올릴 때 누가 접속할 수 있고, 비용이 언제 발생하는가?
데이터 확인 Supabase, SQLite Browser, DBeaver 저장된 데이터와 DB 구조를 확인할 때 데이터가 올바르게 저장되고, 민감한 정보가 안전한가?

처음에는 이 정도면 충분하다

  • 요구사항 정리: Notion 또는 Google Docs
  • 화면 설명: Figma 또는 간단한 손그림
  • 구현 요청: Codex 같은 AI Agent
  • 변경 기록: GitHub
  • 공유 배포: GitHub Pages, Vercel, Netlify 중 하나
  • 데이터 확인: Supabase 또는 SQLite Browser
도구를 많이 쓰는 것이 좋은 프로젝트를 만드는 것은 아니다. 단계별로 필요한 도구를 하나씩 붙이고, 결정 사항을 문서와 Git 기록에 남기는 것이 더 중요하다.

AI Agent에게 말하는 예시

이 프로젝트를 진행하는 데 필요한 도구를 단계별로 추천해줘.

프로젝트 상황:
- 비개발자가 AI Agent와 함께 만든다.
- 사내 업무 도구를 작은 MVP로 시작한다.
- 화면, 데이터 저장, GitHub 저장, 배포가 필요하다.

정리해줄 항목:
- 기획/정리 도구
- 화면 구상 도구
- AI 구현 도구
- Git/GitHub 협업 도구
- 테스트와 검수 도구
- 배포/호스팅 도구
- 데이터 확인 도구

각 도구마다 언제 쓰는지, 비개발자가 확인할 것, 주의할 점을 표로 정리해줘.
