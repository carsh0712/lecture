Section 3-1

3-1 단계. 개발 준비하기

이 단계의 목적

개발 준비는 AI Agent가 바로 코드를 쓰기 전에 방향을 맞추는 단계다. 어떤 문제를 줄이고 싶은지, 누가 사용할지, 어떤 개발 플랫폼이 맞을지 먼저 정리한다.

개발 플랫폼 선택 기준

비전문가가 AI Agent와 함께 작은 업무용 프로그램을 만들 때는 Node.js + Express.js 서버 + React Vite 조합이 가장 무난하다. Express.js는 Node.js에서 서버 기능을 쉽게 만들게 도와주는 도구이고, React Vite는 화면을 빠르게 만들고 확인하기 좋은 도구다.

또한 선택한 플랫폼이 클라우드 서버에서 서비스 가능한지도 함께 확인해야 한다. 로컬에서 잘 동작해도 클라우드 서버가 실행 방법을 이해하지 못하면 팀에 공유할 수 없다.

선택지 언제 적합한가 클라우드 서버 배포 관점 비전문가에게 추천하는 정도
Node.js + Express.js 서버 + React Vite 관리 화면, 검색/필터, 간단한 데이터 조회와 저장이 필요한 웹 프로그램 Node.js 실행과 Dockerfile 구성이 비교적 단순해 배포 예시를 찾기 쉽다. 가장 추천. 화면과 서버를 한 프로젝트 흐름으로 이해하기 쉽다.
Java Spring Boot 회사에 이미 Java 기반 시스템이 있거나, 기존 백엔드 규칙을 따라야 하는 경우 배포는 가능하지만 실행 파일, 메모리, 설정 관리가 더 무거울 수 있다. 팀에 Java 운영 경험이 있을 때 추천. 처음 배우기에는 무거울 수 있다.
Go 빠르고 단순한 서버 프로그램이나 데이터 처리 도구가 필요한 경우 배포 파일은 단순할 수 있지만 팀에 Go 운영 경험이 있는지 확인해야 한다. 개발팀이 Go를 쓰는 경우에 적합. 비전문가가 화면까지 함께 이해하기에는 낯설 수 있다.
특별한 이유가 없다면 처음에는 Node.js + Express.js 서버 + React Vite로 시작하는 것이 좋다. 단, 선택한 방식이 실제 클라우드 서버에서 실행 가능한지도 함께 확인해야 한다.

AI Agent에게 요청하기

우리가 관리하는 사이트 운영을 돕는 프로그램을 만들려고 해.
먼저 개발 준비 단계에서 정해야 할 내용을 정리해줘.

목적은 매일 바뀌는 문의 처리 상태를 빠르게 확인하고 관리하는 것이야.
비전문가가 관리하기 쉬운 기본 선택지로 Node.js + Express.js 서버 + React Vite를 우선 검토해줘.
Java Spring Boot나 Go가 더 나은 경우가 있다면 그 이유도 함께 비교해줘.
각 선택지가 클라우드 서버에서 서비스 가능한지도 함께 확인해줘.
필요한 화면과 데이터, 저장소 준비 항목을 비개발자도 이해할 수 있게 설명해줘.
