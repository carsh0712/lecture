5. AI Agent에게 요청하는 방법

바이브코딩에서 가장 중요한 능력은 코드를 외우는 것이 아니라, AI Agent에게 일을 잘게 나누어 맡기고 결과를 확인하는 것이다. 좋은 요청은 목적, 범위, 확인 기준을 함께 말한다.

요청의 기본 흐름

flowchart TD
  A["목적 말하기
무엇을 만들고 싶은가"]
  B["범위 정하기
이번에 할 것과 하지 않을 것"]
  C["결과 형식 요청
표, 계획, 파일 수정, 체크리스트"]
  D["AI Agent 작업"]
  E["사람이 확인
업무 기준, 화면, API 확인"]
  F{"수정이 필요한가?"}
  G["수정 요청
작게 나누어 말하기"]
  H["완료 기준 확인"]

  A --> B --> C --> D --> E --> F
  F -->|"예"| G --> D
  F -->|"아니요"| H

좋은 요청과 애매한 요청

애매한 요청 좋은 요청 이유
관리 화면 만들어줘. 문의 목록을 보고 상태를 변경하는 내부 관리 화면을 만들어줘. 첫 버전은 등록, 목록, 상태 변경만 포함해줘. 사용 목적과 첫 범위가 분명하다.
DB 붙여줘. 저장할 데이터가 문의 제목, 담당자, 상태, 작성일일 때 SQLite와 PostgreSQL 중 무엇이 맞는지 먼저 비교해줘. 바로 구현하지 않고 판단을 먼저 요청한다.
수정해줘. 목록 화면에서 상태 필터가 위쪽에 보이게 바꾸고, 모바일에서도 버튼이 줄바꿈되게 수정해줘. 수정 위치와 기대 결과가 구체적이다.

요청할 때 포함할 것

  • 누가 사용할지
  • 이번 작업에서 만들 결과물
  • 이번 작업에서 제외할 것
  • AI가 먼저 계획을 세워야 하는지, 바로 파일을 수정해도 되는지
  • 완료 후 어떤 기준으로 확인할지
  • API가 있다면 Swagger/OpenAPI 문서와 테스트 방법이 필요한지

AI Agent에게 말하는 예시

바로 구현하지 말고 먼저 작업 계획을 세워줘.

목표:
- 사내 문의 처리 상태를 관리하는 첫 버전을 만들고 싶다.

이번 범위:
- 문의 등록
- 문의 목록 조회
- 처리 상태 변경

제외할 것:
- 고객 로그인
- 알림 발송
- 복잡한 통계

먼저 해줄 일:
- 필요한 화면 목록
- 필요한 데이터 항목
- 구현 순서
- 내가 확인해야 할 기준
- API가 있다면 Swagger/OpenAPI에서 확인할 요청과 응답
